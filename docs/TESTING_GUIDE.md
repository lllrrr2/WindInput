## 1. 基础输入测试

### 1.1 拼音输入基本流程

| # | 操作 | 预期结果 |
|---|------|---------|
| 1.1.1 | 中文模式下输入 `nihao` | 预编辑区显示 `ni'hao`（带音节分隔符），候选窗出现"你好"等候选 |
| 1.1.2 | 按空格 | 上屏第一个候选词，预编辑区和候选窗消失 |
| 1.1.3 | 输入 `zhongguo` 后按 `1` | 上屏第 1 个候选（通常为"中国"） |
| 1.1.4 | 输入 `beijing` 后按 `2` | 上屏第 2 个候选 |
| 1.1.5 | 输入 `wo` 后按回车 | 上屏原始拼音字母 `wo`（非候选词） |

### 1.2 五笔输入基本流程

|---|------|---------|

| 1.2.2 | 按空格 | 上屏第一个候选 |
| 1.2.3 | 输入 `iiii` 后按回车 | 上屏原始编码 `iiii` |

### 1.3 数字输入

| # | 操作 | 预期结果 |
|---|------|---------|

| 1.3.4 | 先输入 `wo`（有候选），再按 `0` | `0` 不应触发候选选择（仅 1-9 有效），按键透传 |

## 2. 中英文模式切换

### 2.1 基本切换

| # | 操作 | 预期结果 |
|---|------|---------|
| 2.1.1 | 中文模式下按左 Shift（松开时） | 切换到英文模式，显示指示器 "En" |
| 2.1.2 | 英文模式下按左 Shift（松开时） | 切换到中文模式，显示指示器 "中·拼音" 或 "中·五笔" |
| 2.1.3 | 按右 Shift | 同上（默认两个 Shift 都可切换） |

### 2.2 带缓冲区切换

| # | 操作 | 预期结果 |
|---|------|---------|
| 2.2.1 | 输入 `zhong`（有候选），然后按 Shift 切换 | 缓冲区清空，候选窗消失，切换到英文模式 |
| 2.2.2 | 启用 `commit_on_switch`，输入 `zhong`，按 Shift | 原始编码 `zhong` 上屏，切换到英文模式 |
| 2.2.3 | 禁用 `commit_on_switch`，输入 `zhong`，按 Shift | 缓冲区清空（不上屏），切换到英文模式 |

### 2.3 标点跟随模式

| # | 操作 | 预期结果 |
|---|------|---------|
| 2.3.1 | 启用 `punct_follow_mode`，中文模式切到英文 | 标点同时切换为英文标点 |
| 2.3.2 | 上述状态切回中文 | 标点恢复为中文标点 |
| 2.3.3 | 禁用 `punct_follow_mode`，切换中英文 | 标点模式不随之改变 |

### 2.4 模式切换键配置

| # | 操作 | 预期结果 |
|---|------|---------|
| 2.4.1 | 配置 `toggle_mode_keys: [lshift]`，按右 Shift | 不触发切换（仅左 Shift 生效） |
| 2.4.2 | 配置 `toggle_mode_keys: [capslock]`，按 CapsLock | 触发中英文切换 |
| 2.4.3 | 配置 `toggle_mode_keys: [lctrl, rctrl]` | 左/右 Ctrl 均可切换 |

## 3. CapsLock 行为

### 3.1 CapsLock 开启状态

| # | 操作 | 预期结果 |
|---|------|---------|
| 3.1.1 | 中文模式，CapsLock ON，输入 `a` | 直接输出大写 `A`（不进入拼音/五笔） |
| 3.1.2 | CapsLock ON，Shift+`a` | 输出小写 `a` |
| 3.1.3 | CapsLock ON，连续输入字母 | 全部为大写英文字母，无候选窗 |
| 3.1.4 | CapsLock ON 时观察指示器 | 显示大写 "A" 指示 |

### 3.2 CapsLock 关闭状态

| # | 操作 | 预期结果 |
|---|------|---------|
| 3.2.1 | 中文模式，CapsLock OFF，输入字母 | 正常进入中文输入（拼音/五笔） |
| 3.2.2 | CapsLock OFF 时观察指示器 | 显示小写 "a" 指示 |

### 3.3 CapsLock 切换时机

| # | 操作 | 预期结果 |
|---|------|---------|
| 3.3.1 | 中文输入中途（有缓冲区），打开 CapsLock | 缓冲区行为需验证（CapsLock 状态由 C++ Bridge 同步） |
| 3.3.2 | 切换窗口后回来，检查 CapsLock 状态 | 与系统 CapsLock 状态保持同步 |

```yaml
# 五笔专项
engine.wubi.auto_commit_at_4: true/false

```

### 测试矩阵建议

对于组合较多的功能，建议按矩阵测试：

| 维度 | 取值 |
|------|------|
| 引擎 | 拼音 / 五笔 |
| 模式 | 中文 / 英文 |
| 标点 | 中文标点 / 英文标点 |
| 全角 | 全角 / 半角 |
| CapsLock | ON / OFF |

**关键组合**（优先测试）：
- 拼音 + 中文 + 中文标点 + 半角 + CapsLock OFF（最常用场景）
- 五笔 + 中文 + 中文标点 + 半角 + CapsLock OFF
- 拼音 + 中文 + 全角（全角输入验证）
- 五笔 + 中文 + 标点顶字（五笔特殊功能）
- 任意引擎 + CapsLock ON（英文直出验证）
