# 快捷输入（Quick Input）功能设计

## 概述

快捷输入是一种在输入法空码状态下通过触发键（默认分号 `;`）激活的特殊输入模式。该模式下数字键作为输入字符，字母键 a-z 作为候选选择键，支持数字/金额转换、数学计算、日期格式化等功能。

## 触发条件

- 输入法处于中文模式
- 输入缓冲区为空（无未上屏的编码）
- 无候选词
- 按下配置的触发键

## 按键映射

进入快捷输入模式后，按键行为发生以下变化：

| 按键 | 普通模式行为 | 快捷输入模式行为 |
|------|-------------|----------------|
| 数字 0-9 | 候选选择 | 输入字符（追加到缓冲区） |
| 字母 a-z | 编码输入 | 候选选择（a=第1个, b=第2个...） |
| 运算符 +-*/ | 标点/透传 | 输入字符（计算器模式） |
| 点号 . | 标点 | 输入字符（日期/小数模式） |
| 其他符号 | 各自功能 | 输入字符（全部进缓冲区） |
| 空格 | 选首候选 | 选当前高亮候选/重复上屏 |
| 回车 | 上屏编码 | 上屏缓冲区原文 |
| ESC | 取消 | 退出快捷输入 |
| 退格 | 删编码 | 删缓冲区末字符 |
| 上下箭头 | 移动高亮 | 移动高亮 |
| PageUp/Down | 翻页 | 翻页（仅物理键） |
| Z 键 | 编码输入 | 缓冲区空时重复上屏 |

## 功能模块

### 模块 1：重复输入

- **触发**：按触发键后，缓冲区为空时按空格或 Z 键
- **行为**：提取上一次成功上屏的字符串并再次上屏
- **候选显示**：进入模式时自动显示上次上屏内容（无序号标签，只能空格上屏）

### 模块 2：数字与金额转换

- **匹配**：纯数字或小数（如 `12345`、`123.34`）
- **候选列表**（整数）：
  1. 大写金额：壹万贰仟叁佰肆拾伍元整
  2. 小写金额：一万二千三百四十五元整
  3. 小写中文：一万二千三百四十五
  4. 大写中文：壹万贰仟叁佰肆拾伍
  5. 单字小写：一二三四五
  6. 单字大写：壹贰叁肆伍
  7. 千分位：12,345
- **小数金额**（≤2位小数）：支持角、分单位
  - `123.34` → 壹佰贰拾叁元叁角肆分
  - `123.40` → 壹佰贰拾叁元肆角整
  - `123.04` → 壹佰贰拾叁元零肆分
- **小数中文读法**：`123.456` → 一百二十三点四五六

### 模块 3：数学计算器

- **匹配**：包含运算符（+、-、*、/）的表达式，如 `123+456`
- **求值**：递归下降解析器，正确处理乘除优先于加减
- **候选列表**：
  1. 完整算式：123+456=579
  2. 纯结果：579
  3. 结果的大写金额、小写金额、中文等（整数结果时）
- **特殊处理**：
  - 尾部运算符自动忽略（`123+` 不计算）
  - 除零返回错误（不生成候选）
  - 小数精度可配置（默认 6 位，0 表示取整）

### 模块 4：日期格式化

- **年月日**（三段）：`2026.4.2`
  1. 紧凑格式：20260402
  2. 中文格式：2026年4月2日
  3. 带前导零：2026年04月02日
  4. ISO 格式：2026-04-02
  5. 斜杠格式：2026/04/02
- **年月**（两段，首段>31）：`2026.3`
  1. 中文格式：2026年3月
  2. 带前导零：2026年03月
  3. 连字符：2026-03
  4. 斜杠：2026/03

### 候选合并

同一输入可能匹配多个模块（如 `2026.03` 既是小数也是年月），系统会合并所有匹配模块的候选并去重。路由优先级：日期 > 年月 > 计算器 > 数字。

## UI 适配

### 候选序号标签

快捷输入模式下，候选序号从数字（1/2/3...）切换为字母（a/b/c...），通过 `Candidate.IndexLabel` 字段实现，渲染器优先使用该字段。

### 强制竖排

可配置进入快捷输入时强制切换为竖排布局（默认开启），退出后自动恢复原布局。

### 动态窗口宽度

竖排模式下，窗口宽度根据候选文本长度动态调整（最小 280px，最大 600px），超长文本使用省略号（…）截断。

### 右键菜单

快捷输入模式下，右键菜单中前移/后移/置顶/删除/恢复默认等操作禁用（灰色），仅保留"复制"可用。

## 配置项

```yaml
input:
  quick_input:
    enabled: true           # 总开关
    trigger_key: semicolon  # 触发键
    force_vertical: true    # 强制竖排显示
    decimal_places: 6       # 小数保留位数（0=取整）
```

### 支持的触发键

| 配置值 | 按键 |
|--------|------|
| semicolon | ; |
| backtick | ` |
| quote | ' |
| comma | , |
| period | . |
| slash | / |
| backslash | \ |
| open_bracket | [ |
| close_bracket | ] |

### 冲突处理

- 快捷输入仅在空码时触发，不与候选选择键冲突
- 临时拼音触发键检查在快捷输入之前，优先级更高
- 设置界面会提示与临时拼音的按键冲突

## 退出条件

- 按 ESC 键
- 退格键将缓冲区删空后再按退格
- 完成一次候选上屏后
- 缓冲区为空时再次按触发键（上屏触发键字符）
- 模式切换、焦点丢失等外部事件（通过 clearState）

## 文件结构

| 文件 | 职责 |
|------|------|
| `coordinator/handle_quick_input.go` | 核心框架：进入/退出、按键路由、候选生成、UI 显示 |
| `coordinator/quick_input_number.go` | 数字/金额转换（整数+小数） |
| `coordinator/quick_input_calc.go` | 计算器（递归下降解析器） |
| `coordinator/quick_input_date.go` | 日期格式化（年月日+年月） |
| `candidate/candidate.go` | IndexLabel 字段 |
| `ui/renderer_layout.go` | IndexLabel 渲染、动态宽度、文本截断 |
| `ui/window.go` + `window_mouse.go` | 右键菜单快捷输入模式支持 |
| `config/config.go` | QuickInputConfig 配置 |
