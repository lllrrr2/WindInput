# 词库补丁 (Dict Patch)

## 概述

词库补丁机制允许在**不修改原始词库文件**的情况下，对词条进行新增、删除、调整权重等操作。适用于使用第三方词库（如 rime-jidian）但需要局部定制的场景。

补丁在编译词库缓存（wdb/wdat）时自动发现并应用，**零运行时开销**。

## 补丁文件

### 位置与命名

补丁文件与目标词库文件**同目录**，命名规则：

```
<词库名>.dict.yaml       ← 原始词库
<词库名>.dict.patch.yaml ← 对应的补丁文件
```

示例：

| 词库文件 | 补丁文件 |
|---------|---------|
| `wubi86_jidian.dict.yaml` | `wubi86_jidian.dict.patch.yaml` |
| `wubi86_jidian_extra.dict.yaml` | `wubi86_jidian_extra.dict.patch.yaml` |
| `rime_ice.dict.yaml` | `rime_ice.dict.patch.yaml` |

每个词库文件可以有独立的补丁，互不干扰。主词库和 `import_tables` 中声明的关联词库都支持补丁。

### 格式

补丁文件为 YAML 格式，包含两个可选段：

- **`entries`** — 新增或修改词条（code+text 已存在则更新权重，否则新增）
- **`delete`** — 删除词条（按 code+text 精确匹配）

字段名支持**全称和缩写**两种写法：

| 全称 | 缩写 | 说明 |
|------|------|------|
| `code` | `c` | 编码（五笔编码或拼音无空格拼接） |
| `text` | `t` | 词条文字 |
| `weight` | `w` | 权重（数值越大优先级越高） |
| `pinyin` | `p` | 拼音，空格分隔音节（可选，仅拼音词库用于更新简拼索引） |

### 示例

```yaml
---
# 缩写写法（推荐，搭配 YAML flow 语法，一行一条）
entries:
  - {c: a, t: 工, w: 30}         # 修改「工」在编码 a 下的权重为 30
  - {c: abcd, t: 自定义词, w: 100} # 新增词条

delete:
  - {c: a, t: 戈}                 # 删除编码 a 下的「戈」
```

```yaml
---
# 全称写法（等价）
entries:
  - code: a
    text: 工
    weight: 30
  - code: abcd
    text: 自定义词
    weight: 100

delete:
  - code: a
    text: 戈
```

两种写法可以混用。

### 拼音词库补丁

拼音词库的补丁在新增词条时，可通过 `pinyin`/`p` 字段指定音节，自动更新简拼索引：

```yaml
entries:
  - {c: nihao, t: 你好, w: 2000, p: "ni hao"}
```

如果不提供 `pinyin` 字段，词条仍会被新增到主索引，但不会出现在简拼匹配结果中。

## 工作原理

```
原始词库 (.dict.yaml)
        │
        ▼
   加载到内存 (map[code][]entry)
        │
        ▼
   发现补丁文件 (.dict.patch.yaml)  ← 自动按命名规则查找
        │
        ▼
   应用补丁（delete → entries）     ← 先删后增，避免冲突
        │
        ▼
   排序 + 写入二进制缓存 (.wdb/.wdat)
```

### 缓存联动

补丁文件纳入了缓存失效检测的源文件列表。当补丁文件**新增、修改或删除**时，下次启动会自动重新编译词库缓存，无需手动清理。

## 注意事项

1. **删除操作先于新增执行**：如果对同一 code+text 同时写了 delete 和 entries，会先删再加，最终结果是以 entries 中的权重新增。

2. **权重值参考原始词库**：修改权重时，建议先查看原始词库中同编码下其他词条的权重，确保调整后的相对顺序符合预期。

3. **特殊字符需加引号**：如果词条文字包含 YAML 特殊字符（`: # [ ] { }`），需用引号包裹：
   ```yaml
   entries:
     - {c: zz, t: "::", w: 10}
   ```

4. **补丁不会修改原始词库文件**：所有变更仅影响编译后的二进制缓存，原始 `.dict.yaml` 文件保持不变。

5. **空补丁文件无副作用**：补丁文件存在但内容为空（或全部注释）时，不会影响词库编译。
