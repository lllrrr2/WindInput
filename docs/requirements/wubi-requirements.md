# 五笔输入法需求文档

## 1. 文档说明

- **来源**: `D:\Download\Develop\Wubi\五笔输入法功能汇总.pdf`
- **整理时间**: 2026-03-10
- **整理方式**: 基于 PDF 中“六款五笔输入法 30 项功能对比”和“我的习惯”列进行归纳。

本文档用于沉淀用户对五笔输入法的核心功能诉求，作为后续产品设计、实现排期和验收测试的依据。  
注意：原始 PDF 同时包含“竞品能力对比”和“个人使用偏好”，因此本文档将需求分为“必须支持”“建议支持”“可选增强”“默认关闭/暂不需要”四类，而不是简单照搬对比表。

## 2. 产品定位

目标产品应以**纯五笔输入体验**为核心，优先保证：

1. 基础输入行为可配置，且配置项符合成熟五笔用户习惯。
2. 词库可维护、可迁移、可导入导出，不把用户锁死在单机环境中。
3. 高频操作尽量通过快捷键完成，避免必须进入设置界面。
4. 默认行为尽量贴近五笔老用户习惯，对“混输”“联想”“自动提示”类能力保持克制。

## 3. 需求分级

### 3.1 P0 必须支持

| 编号 | 需求 | 说明 |
|------|------|------|
| R01 | 词库导入/导出 | 必须支持词库导入导出，且系统词库应支持多种类型的词条，便于在单一平台维护后分发到其他环境。 |
| R02 | 自定义标点 | 必须支持自定义标点映射。 |
| R03 | `Shift + 字母` 行为可配置 | 必须支持在“大写字母直接上屏”和“转入临时英文”之间切换。 |
| R04 | 数字后句号行为可配置 | 必须支持在“直接输出句号”和“自动转为小数点”之间切换。 |
| R05 | 四码唯一自动上屏可配置 | 必须支持开启或关闭“四码唯一自动上屏”。 |
| R06 | 第五码顶字上屏可配置 | 必须支持开启或关闭“五码顶字上屏”。 |
| R07 | 纯五笔模式 | 必须具备纯五笔模式，不能仅提供混输模式。 |
| R08 | 候选数量可配置 | 候选项个数应允许灵活调整，建议覆盖 3 至 10；若支持 `0` 选第 10 项更佳。 |
| R09 | 分号/引号选重 | 必须支持分号、引号作为二三候选快捷键，且可作为默认方案。 |
| R10 | 中英文切换键 | 必须支持使用 `Shift` 切换中英文；若可单独设为左 `Shift` 更符合用户习惯。 |
| R11 | 切换到英文时编码处理 | 中文组字过程中切到英文时，当前编码必须支持直接上屏，且应作为默认行为。 |
| R12 | 临时拼音/临时生僻字引导 | 必须提供临时拼音和/或临时生僻字输入入口，至少要有明确快捷键方案。 |
| R13 | 回车键行为 | 回车键必须支持“编码上屏”，并允许按需配置为“清空编码”或“编码上屏”。 |
| R14 | `z` 键重复录入 | 必须支持 `z` 键重复录入，并可默认开启。 |
| R15 | 单候选时选重键行为 | 当候选窗仅剩一项时，分号/引号最好不起作用，避免误触顶字。 |

### 3.2 P1 建议支持

| 编号 | 需求 | 说明 |
|------|------|------|
| R16 | 四码空码处理策略 | 建议支持“保留候选窗 / 清空候选窗 / 转临时英文”等策略，默认更贴近“保留候选窗”。 |
| R17 | 五笔拼音混输 | 在纯五笔模式之外，建议支持混输模式；纯拼音模式可作为扩展能力。 |
| R18 | 快捷键加词 | 建议提供快捷键直接加词，减少进入设置界面的频率。 |
| R19 | 快捷键删词 | 建议支持对候选项直接删词，并明确删除结果是“修改系统词库”还是“加入屏蔽列表”。 |
| R20 | 快捷键调词序 | 建议支持通过快捷键调词序，且排序结果最好可导出、可迁移。 |
| R21 | 生僻字词开关 | 建议提供生僻字词输出开关，用于控制低频字词是否进入常规候选。 |
| R22 | 生僻字词自定义 | 建议支持自定义标记生僻字词，而不只是固定依赖系统词频。 |
| R23 | 候选窗样式 | 建议至少支持单行样式；双行、多行可作为额外样式。用户偏好是单行。 |
| R24 | 编码显示位置 | 建议支持在候选窗内显示编码；这是用户的首选形式。 |

### 3.3 P2 可选增强

| 编号 | 需求 | 说明 |
|------|------|------|
| R25 | 输入模式扩展 | 可扩展到纯拼音模式、更多拼音方案、更多形码方案。 |
| R26 | 词库维护平台化 | 可进一步区分系统词库、用户词库、自定义短语，并提供互转、同步或统一维护能力。 |
| R27 | 生僻字词标记快捷键 | 可支持通过快捷键或鼠标右键将候选标记为生僻字词。 |
| R28 | 候选数与快捷键深度自定义 | 可允许用户自定义候选数、候选键、翻页键等完整键位组合。 |

### 3.4 默认关闭或暂不需要

以下能力可不作为默认开启项，甚至可以不进入近期排期：

| 编号 | 需求 | 说明 |
|------|------|------|
| N01 | 编码逐渐提示 | 用户偏好关闭。 |
| N02 | 词组联想 | 用户偏好关闭。 |
| N03 | 标点自动配对 | 用户偏好关闭。 |
| N04 | 空码时检索生僻字 | 用户偏好关闭。 |
| N05 | 临时英文快捷键 | 用户明确表示基本不用此能力。 |
| N06 | 简入繁出 | 当前无明确需求。 |

## 4. 关键交互要求

### 4.1 输入行为

1. 纯五笔输入必须是第一优先级体验，不能因混输能力破坏基本键感和组字逻辑。
2. 自动上屏、顶字、空码处理都必须是可配置项，不能强制绑定单一行为。
3. 用户从中文切到英文时，当前编码默认应上屏，不应直接丢失。
4. 回车键默认应更偏向“编码上屏”，因为这更符合该用户的习惯。

### 4.2 快捷键体系

1. `Shift` 是核心模式切换键，最好支持细分为左 `Shift`、右 `Shift`。
2. 分号/引号选重应作为默认二三候选方案保留。
3. 加词、删词、调词序这三类高频词库维护动作，建议全部提供快捷键。
4. 临时拼音、生僻字引导需要独立快捷键，且允许和引擎切换键分开配置。

### 4.3 词库与数据维护

1. 系统词库必须可导入导出。
2. 词库格式应尽量兼容多种词条类型，避免对非汉字、特殊编码、人为约定前缀做过度限制。
3. 用户通过快捷键形成的词库调整，最好能沉淀为可迁移的数据，而不是只存在本机内存状态。

## 5. 默认配置建议

基于 PDF 中“我的习惯”列，建议默认配置倾向如下：

| 配置项 | 建议默认值 |
|------|------|
| 输入模式 | 纯五笔 |
| 四码唯一自动上屏 | 关闭，由用户决定 |
| 第五码顶字上屏 | 关闭，由用户决定 |
| 四码空码处理 | 保留候选窗 |
| 候选数量 | 5 至 9 之间任选，建议保守默认值 |
| 二三候选键 | 分号 / 引号 |
| 中英文切换 | `Shift` |
| 切英文时处理 | 编码上屏 |
| 回车键行为 | 编码上屏 |
| 候选窗样式 | 单行 |
| 编码显示位置 | 候选窗内 |
| `z` 键重复录入 | 开启 |
| 编码提示 | 关闭 |
| 词组联想 | 关闭 |
| 标点自动配对 | 关闭 |
| 空码检索生僻字 | 关闭 |
